<objective>
Build the teacher dashboard: class overview, per-student progress visibility, and figure assignment.

Purpose: Enable structured classroom use with teacher oversight.
Output: Role-gated teacher pages, data fetching, assignment flow.
</objective>

<context>
- profiles have a role field defaulting to "student"
- user_progress tracks exercises per student per figure
- writing_submissions stores student writing samples
- figures catalog has 20 figures with categories and difficulty
- Existing progress components from Phase 3 can be reused
</context>

  <name>Add teacher schema tables and migration</name>
  <files>src/db/schema.ts, drizzle/0002_phase4_teacher.sql</files>
  <action>
    1. Add `classes` table: id (serial PK), name (varchar), teacherId (uuid FK to profiles), createdAt, updatedAt
    2. Add `class_students` table: id (serial PK), classId (FK to classes, cascade), studentId (uuid FK to profiles, cascade)
    3. Add `assignments` table: id (serial PK), classId (FK to classes, cascade), figureId (FK to figures), assignedBy (uuid FK to profiles), dueDate (timestamp, optional), createdAt
    4. Add Drizzle relations
    5. Generate migration SQL with foreign keys and cascades
    6. Export inferred types
  </action>
</task>
